Extra credit
------------
Will use [code](20240209.hs) to count number of possible puzzles for each
loop.

The upper bound for each loop is 2⁹ = 512.  Obviously no numbers
present is not a puzzle, is 511 also an upper bound.

Using brute force, iterate over all 512 sets of numbers,
and for each of those, iterate over the 212 other possible loops and it
will be a puzzle if none of them are consistent with the set of numbers.

And the brute force calculation is pretty slow, but it counts 41433 possible
four-by-four/three-by-three puzzles.

Making the rounds
-----------------
The top block has √(ab³c³d) = bc√(abcd), so abcd must be a perfect square.

To make the smallest number, make b and c the smallest prime numbers, 2 and
3.  Then making a and d 2×4 and 3×4 makes the top number
2×3×√(2×4×2×3×3×4) = 144.
